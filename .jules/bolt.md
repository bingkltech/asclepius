## 2024-04-22 - Optimized App.tsx array lookups inside list mapping
**Learning:** O(N) array lookups (`workers.find`) were nested inside O(M) component mapping functions (`dagTasks.map`) within the primary UI render path in `App.tsx`. Because `workers` and `dagTasks` scales with usage, this pattern causes an O(N*M) rendering complexity drop. The fix is applying `useMemo` specifically outside the loop context to establish an O(1) hash map lookup reference (`workerMap.get`). This prevents unnecessary nested list scanning on every frame render.
**Action:** Always hoist complex derivations like array lookups out of list mappers/filters within React components and use `useMemo` when rendering dynamic lists. Look for `.find()` inside `.map()` as a key performance anti-pattern.
