## 🔴 HIGH PRIORITY — Context Pipeline Refactor (Bring Back)
The current "Ping Mode" in CommandCenter.tsx is a BANDAID. 
The full context pipeline needs a proper tiered intent classification system:
  - Every chat message currently triggers a full LLM call (even "hi")
  - Ollama context window (8192 tokens) can overflow with long conversations
  - The system has no message intent detection (greeting vs. task vs. query)
  - No caching of system state between rapid messages
  
  Proposed future architecture:
  1. Intent Classifier (local, no LLM): categorize message as ping/greeting, 
     status query, task command, or complex analysis
  2. Tiered Context Loading: ping=minimal, status=agents only, task=full context
  3. Streaming responses for Ollama (stream: true) to improve perceived speed
  4. Context caching: don't rebuild agent/project context if nothing changed
  5. History windowing: sliding window with summarization for long conversations
